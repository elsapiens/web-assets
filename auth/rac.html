RBAC Multi-Tenant Auth System - Role & Capability Flow

Purpose: Multi-tenant permission management and access control for applications in a reseller-based authentication system with SSO support.

Key Concepts

Module
Sections of an application (e.g., createuser, holidays, billing)
Capability
Specific actions within a module (e.g., read, write, update, delete)
Reseller
Partner who sells the company's software under their brand in exchange for commission. Has SSO access and can have their own workspaces.
Workspace
Client area where the client and their employees manage various apps
Role
Group of capabilities that can be assigned to users
Person Additional Capabilities
Optional direct capability assignments to individuals, bypassing roles

Typical Flow

Admin creates role
Admin assigns capabilities to role
Admin assigns role to person
Person can perform actions based on assigned capabilities
App Registry Application

Central registry of all application modules and their capabilities

Table: module
Column Type Description
id UUID Primary key
app_id UUID Application identifier
name String Module name (e.g., "users", "holiday", "billing")
description Text Module purpose description
UNIQUE (app_id, name)
Example Records:
id app_id name description
uuid-001 app-hrms-001 users User management module
uuid-002 app-hrms-001 holiday Holiday tracking module
uuid-003 app-hrms-001 payroll Payroll processing module
Table: capability
Column Type Description
id UUID Primary key
module_id UUID Foreign key to module table
name String Capability name (e.g., "read", "write", "delete")
description Text Action description
UNIQUE (module_id, name)
Example Records:
id module_id name description
cap-001 uuid-001 read View user information
cap-002 uuid-001 write Create new users
cap-003 uuid-001 update Modify user details
cap-004 uuid-002 read View holiday calendar
Elauth Application

Multi-tenant authentication system managing roles and permissions per workspace/reseller

Table: roles
Column Type Description
id UUID Primary key
reseller_id UUID Reseller who owns this role
workspace_id UUID Workspace where role applies
name String Role name (e.g., "Admin", "Manager", "Employee")
description Text Role purpose description
UNIQUE (reseller_id, workspace_id, name)
Example Records:
id reseller_id workspace_id name description
role-001 reseller-alpha ws-acme-corp HR Admin Full access to HR system
role-002 reseller-alpha ws-acme-corp Employee Basic employee access
Table: role_capability
Column Type Description
id UUID Primary key
role_id UUID Foreign key to roles table
app_id UUID Application identifier
capabilities JSON Map of module names to capability arrays
UNIQUE (role_id, app_id)
Example Records:
id role_id app_id capabilities (JSON)
rc-001 role-001 app-hrms-001 
{
  "users": ["read", "write", "update"],
  "holiday": ["read", "write"],
  "payroll": ["read", "write", "update"]
}
rc-002 role-002 app-hrms-001 
{
  "users": ["read"],
  "holiday": ["read"]
}
Table: person_role_assignment
Column Type Description
id UUID Primary key
workspace_id UUID Workspace context
reseller_id UUID Reseller context
person_id UUID User being assigned the role
role_id UUID Role being assigned
UNIQUE (workspace_id, person_id, role_id)
Example Records:
id workspace_id reseller_id person_id role_id
pra-001 ws-acme-corp reseller-alpha person-john-doe role-001
pra-002 ws-acme-corp reseller-alpha person-jane-smith role-002
Table: person_additional_capabilities
Column Type Description
id UUID Primary key
workspace_id UUID Workspace context
reseller_id UUID Reseller context
person_id UUID User receiving additional capability
capability_id UUID Specific capability being granted
UNIQUE (workspace_id, person_id, capability_id)
Example Records:
id workspace_id reseller_id person_id capability_id
pac-001 ws-acme-corp reseller-alpha person-jane-smith cap-003

Example: Jane Smith (Employee role) gets additional "update" capability on "users" module
